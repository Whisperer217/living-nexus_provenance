# Living Nexus Architectural Drift Protocol (LN-ADP v1)

## Purpose

This protocol exists to prevent architectural, philosophical, and user-experience drift as Living Nexus evolves.

Before implementing any new feature, system, page, workflow, AI integration, monetization layer, or creator experience, evaluate it against this document.

The goal is not maximum features.

The goal is maintaining coherence.

---

# Core Principle

Living Nexus is a Creator Domain Platform.

Everything else is secondary.

The primary object of the platform is:

Creator Domain

Not:

* Tracks
* Users
* Albums
* Comics
* Stores
* AI tools

All manifestations belong to a Creator Domain.

Every feature must strengthen the Creator Domain.

---

# The Five North Stars

## 1. Identity Before Upload

A creator should establish identity before manifestations.

Ask:

"Does this feature strengthen creator identity?"

If not, reconsider.

---

## 2. Domain Before Feed

A creator domain is more important than any homepage feed.

Ask:

"Does this improve the creator's owned space?"

If not, reconsider.

---

## 3. Discovery Through Exploration

Living Nexus is not an infinite-scroll platform.

Living Nexus is a discovery platform.

Ask:

"Does this create exploration or passive consumption?"

Prefer exploration.

---

## 4. Provenance Over Virality

Popularity is useful.

Provenance is essential.

Ask:

"Does this preserve origin, continuity, witness, and attribution?"

If not, reconsider.

---

## 5. Exportability Over Dependency

Creators must be able to leave without losing themselves.

Ask:

"Can this data be exported?"

If not, redesign.

---

# Creator Domain Test

Before merging any feature:

Can this be explained as improving one of:

* Identity
* Manifestations
* Shelves
* Provenance
* Distribution
* Artifacts
* Domain History

If not, it may not belong.

---

# Shelf Rule

Collections should be experienced as shelves, racks, archives, displays, and exhibits.

Avoid reducing everything to lists.

Prefer:

* Music Shelf
* Comic Rack
* Book Shelf
* Artifact Case

Over:

* Generic Grid
* Generic Table
* Generic Feed

---

# AI Rule

AI may assist:

* organization
* metadata
* discovery
* packaging
* continuity
* transformation

AI must not replace:

* creator identity
* testimony
* provenance
* authorship

Humans originate meaning.

---

# Trust Restoration Rule

No major feature work while critical paths are broken.

Critical paths include:

* Identity
* Domain
* Upload
* Archive
* Playback
* QR Resolution
* Distribution

Trust takes priority over expansion.

---

# Physical Reality Rule

Living Nexus ultimately bridges digital manifestations into physical artifacts.

Every major system should remain compatible with:

* Books
* Comics
* CDs
* Vinyl
* USB Archives
* Certificates
* Future physical media

Avoid cloud-only assumptions.

---

# The Cosmos Rule

The Cosmos is emergent.

Do not build visual complexity before relationship complexity exists.

Build:

Creator → Manifestation → Provenance → Discovery Trail

First.

The map comes later.

---

# Drift Check

Before every phase ask:

1. Does this strengthen Creator Domains?
2. Does this improve discovery?
3. Does this preserve provenance?
4. Does this maintain exportability?
5. Does this increase trust?
6. Does this move manifestations toward physical reality?
7. Would this still make sense if social media disappeared?

If four or more answers are "No", stop and reassess.

---

# Final Principle

Living Nexus is not a social network.

It is a network of creator-owned domains connected through provenance, discovery, and manifestation.

Features may evolve.

This principle should not.
